Detailed Description

A driver class for Romi's IMU, BNO.

BNO.py contains the class driver for the BNO055 intertial measurement unit. It checks if a calibration file exists already on the robot's flash memory, or else it triggers the calibration sequence and creates one. Like all of Romi's firmware, it initializes with a set of Shares designed to pass important data in and out to the rest of Romi's program.

Since BNO communicates with Romi over I2C, it also initializes with an I2C object already set up for communication.

Class initializes a BNO object, handles calibration, communivation, and regularly updates Shares with IMU data.

Initializes and returns an object associated with a BNO IMU.

BNO unpacks its Shares and creates variables to use for data. It also creates variables containing all of BNO's relevant memory addresses, for reference. 

BNO_shares contains nine Shares. They are used to pass out data
and to communicate with the main program.
BNO_shares[0] = heading     [rad]   (float)
BNO_shares[1] = calibration complete(flag)
BNO_shares[2] = eul_x       [rad]   (float)
BNO_shares[3] = eul_y       [rad]   (float)
BNO_shares[4] = eul_z       [rad]   (float)
BNO_shares[5] = x_ang_vel   [rad/s] (float)
BNO_shares[6] = y_ang_vel   [rad/s] (float)
BNO_shares[7] = z_ang_vel   [rad/s] (float)
BNO_shares[8] = req zero heading    (flag)
Parameters
BNO_sharesA tuple containing all of BNO's Shares objects.
i2cAn I2C object already set up to communicate with BNO.

◆ MainTask()

BNO.BNO.MainTask (   self)

Main cotask task for BNO.

The BNO main task has states: 

    1:  ???
    2:  ???
    3:  ???
    4:  ???
    5:  Normal operation state. Continuously read IMU registers and push
        data to corresponding Shares. Performs a short calculation for phi
        which accounts for zeroing out the IMU euler x angle.

Like all of Romi's cooperative multitasking tasks, MainTask is written as a generator function with an infinite loop. Each pass through the program runs through the MainTask while loop once, until it reaches a yield.

◆ zero_phi()

BNO.BNO.zero_phi (   self)

A helper function used to zero out the phi parameter (Romi heading)

zangle is the z-offset used to calculate Romi's heading. On startup, Romi assumes that the initial position is X,Y (0,0) and heading 0. BNO only speaks in offset from what it measures to be true north. zangle is then the difference between true north and the initial heading, and just needs to be equal to the IMU euler x angle at the desired zero position.
